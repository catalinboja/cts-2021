1. During the game the client may lose the connection to the server (for various reasons). When this happens,
the different modules in the game must react accordingly
- the game instance must be saved on the client
- character attributes must be saved
- the gamer must be notified
- the connection must be retried


2. Players have the opportunity to save the state of the game / character and restore it later
- the player can resume the game from a certain point
